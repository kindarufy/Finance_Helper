# Тестовые сценарии для Finance Helper

Ниже приведён полный набор сценариев ручного тестирования для проекта **Finance Helper**. 
Сценарии покрывают Telegram-бота, Mini App, экспорт, импорт выписки, категории, лимиты, отчёты, workspaces, команды, UX и негативные случаи.

## 0. Подготовка окружения
**Цель:** убедиться, что система запускается на облачном сервере и доступна для проверки.

**Шаги:**
1. Создать `.env` на основе `.env.example` и заполнить реальные значения переменных окружения.
2. Проверить доступность PostgreSQL.
3. Выполнить миграции финансового сервиса: `alembic upgrade head`.
4. Запустить `finance-service`, `analytics-service`, `api-gateway` и `bot-service` как отдельные серверные процессы.
5. Проверить health endpoints сервисов.
6. Проверить, что `MINIAPP_PUBLIC_URL` указывает на постоянный HTTPS-домен, а не на ngrok.

**Ожидаемый результат:**
- база данных доступна;
- миграции проходят успешно;
- `finance-service`, `analytics-service`, `api-gateway` и `bot-service` запускаются;
- Telegram-бот отвечает пользователю;
- Mini App открывается по домену без запуска ngrok;
- в логах нет traceback и критических ошибок.
