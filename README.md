# Релиз-версия программного продукта
## «Учёт личных финансов с использованием Telegram-бота»

Данный репозиторий содержит релиз-версию программного продукта **Finance Helper**, разработанного в рамках выпускной квалификационной работы.

Программный продукт предназначен для учёта, контроля и анализа личных финансов пользователя с использованием Telegram-бота, Mini App и микросервисной архитектуры.

---

## 1. Назначение программного продукта

**Finance Helper** — это программный продукт для ведения личного и совместного бюджета в Telegram.

Система позволяет пользователю:
- фиксировать доходы и расходы;
- управлять категориями, лимитами и рабочими пространствами;
- просматривать операции и отчёты;
- анализировать финансовую активность;
- работать с данными через Telegram-бота и Mini App.

---

## 2. Основные возможности системы

Релиз-версия программного продукта позволяет:

- вести учёт доходов и расходов;
- добавлять операции текстом, командами и кнопками меню;
- указывать дату операции, в том числе задним числом;
- просматривать, редактировать и удалять операции;
- создавать собственные категории доходов и расходов;
- назначать ключевые слова для автоматического подбора категории;
- устанавливать лимиты и бюджеты;
- получать уведомления при достижении лимитов;
- формировать отчёты за период;
- получать дневную финансовую сводку;
- выполнять аналитический анализ трат;
- использовать совместные бюджеты и общие пространства;
- экспортировать данные в CSV и XLSX;
- открывать Mini App для просмотра расширенной аналитики;
- импортировать банковские выписки.

---

## 3. Архитектура проекта

Программный продукт реализован на основе микросервисной архитектуры и включает следующие сервисы:

### `finance-service`
Сервис хранения и обработки финансовых данных.

Отвечает за пользователей, операции, категории, лимиты, рабочие пространства, участников общих бюджетов и импорт выписок. Использует базу данных PostgreSQL.

### `analytics-service`
Сервис аналитики и формирования отчётов.

Отвечает за дневные сводки, месячные отчёты, анализ расходов, экспорт данных в CSV/XLSX и данные для Mini App.

### `api-gateway`
Единая точка входа для внутренних и пользовательских запросов. Отвечает за маршрутизацию запросов между сервисами и отдачу Mini App.

### `bot-service`
Telegram-бот, предоставляющий основной пользовательский интерфейс.

### `Mini App`
Веб-интерфейс для расширенного просмотра данных пользователя. В актуальной версии Mini App открывается по постоянному HTTPS-домену, поэтому `ngrok` не требуется.

Взаимодействие между сервисами осуществляется по HTTP с использованием внутреннего API-ключа.

---

## 4. Используемые технологии

В проекте использованы следующие технологии:

- Python 3.11+;
- FastAPI;
- aiogram;
- PostgreSQL;
- SQLAlchemy;
- Alembic;
- Pydantic;
- REST API;
- HTML, CSS, JavaScript;
- Telegram Bot API;
- Telegram Mini App;

## 6. Настройка окружения

Перед запуском проекта необходимо перейти в каталог `finance_helper/source_files`, создать файл `.env` на основе шаблона `.env.example` и указать реальные значения переменных окружения.

Обязательные параметры:

- `BOT_TOKEN` — токен Telegram-бота;
- `INTERNAL_API_KEY` — внутренний ключ взаимодействия сервисов;
- `MINIAPP_SIGNING_SECRET` — секретный ключ для подписи токенов Mini App;
- `MINIAPP_PUBLIC_URL` — публичный HTTPS-адрес Mini App на домене;
- параметры подключения к базе данных PostgreSQL;
- `FINANCE_URL`, `ANALYTICS_URL`, `GATEWAY_URL` — адреса сервисов на сервере.

Пример подготовки файла окружения:

```bash
cd finance_helper/source_files
cp .env.example .env
```

Файл `.env` содержит секретные данные и не должен публиковаться в GitHub или передаваться в составе релизного архива.

## 7. Установка зависимостей

Для серверного запуска используется виртуальное окружение Python.

Создание и активация окружения:

```bash
cd finance_helper/source_files
python3 -m venv venv
source venv/bin/activate
```

Установка зависимостей:

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

Если зависимости устанавливаются по сервисам отдельно, можно использовать файлы `requirements.txt` внутри каталогов сервисов.

## 8. Подготовка базы данных

Перед запуском сервисов необходимо создать базу данных PostgreSQL и пользователя, указанных в `.env`. После этого выполняются миграции финансового сервиса:

```bash
cd finance_helper/source_files/services/finance-service
alembic upgrade head
```

## 9. Запуск проекта на сервере

Сервисы запускаются как отдельные Python-процессы. Ниже приведены команды для ручного запуска. Для постоянной эксплуатации на сервере рекомендуется оформить эти команды как `systemd`-службы или использовать другой менеджер процессов.

### 9.1. Запуск `finance-service`

```bash
cd finance_helper/source_files/services/finance-service
uvicorn app.main:app --host 0.0.0.0 --port 8001
```

### 9.2. Запуск `analytics-service`

```bash
cd finance_helper/source_files/services/analytics-service
uvicorn app.main:app --host 0.0.0.0 --port 8002
```

### 9.3. Запуск `api-gateway`

```bash
cd finance_helper/source_files/services/api-gateway
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 9.4. Запуск `bot-service`

```bash
cd finance_helper/source_files/services/bot-service
python -m app.main
```

При серверном запуске Telegram-бот обращается к API Gateway через переменную `GATEWAY_URL`. Mini App открывается по адресу, указанному в `MINIAPP_PUBLIC_URL`.

---

## 10. Mini App и домен

В актуальной версии проекта Mini App привязан к постоянному HTTPS-домену. Для этого в `.env` указывается:

```env
MINIAPP_PUBLIC_URL=https://your-domain.example/miniapp/app
```

Домен должен быть настроен так, чтобы запросы к `/miniapp/app` и `/miniapp/public/...` попадали в `api-gateway`. Запуск `ngrok` для демонстрации Mini App больше не требуется.

## 12. Тестирование

Автоматизированные тесты находятся в каталоге:

```text

```

Запуск тестов:

```bash
cd finance_helper/source_files
PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 python -m pytest -q
```

Часть smoke-тестов проверяет доступность запущенного `api-gateway`. Если gateway не запущен, такие тесты будут пропущены.
