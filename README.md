# Релиз-версия программного продукта
## «Учёт личных финансов с использованием Telegram-бота»

Данный репозиторий содержит релиз-версию программного продукта **Finance Helper**, разработанного в рамках выпускной квалификационной работы.

Программный продукт предназначен для учёта, контроля и анализа личных финансов пользователя с использованием Telegram-бота, Mini App и микросервисной архитектуры.

---

## 1. Назначение программного продукта

**Finance Helper** — это программный продукт для ведения личного и совместного бюджета в Telegram.

Система позволяет пользователю:
- фиксировать доходы и расходы;
- управлять категориями, лимитами и рабочими пространствами;
- просматривать операции и отчёты;
- анализировать финансовую активность;
- работать с данными через Telegram-бота и Mini App.

---

## 2. Основные возможности системы

Релиз-версия программного продукта позволяет:

- вести учёт доходов и расходов;
- добавлять операции текстом, командами и кнопками меню;
- указывать дату операции, в том числе задним числом;
- просматривать, редактировать и удалять операции;
- создавать собственные категории доходов и расходов;
- назначать ключевые слова для автоматического подбора категории;
- устанавливать лимиты и бюджеты;
- получать уведомления при достижении лимитов;
- формировать отчёты за период;
- получать дневную финансовую сводку;
- выполнять аналитический анализ трат;
- использовать совместные бюджеты и общие пространства;
- экспортировать данные в CSV и XLSX;
- открывать Mini App для просмотра расширенной аналитики;
- импортировать банковские выписки.

---

## 3. Архитектура проекта

Программный продукт реализован на основе микросервисной архитектуры и включает следующие сервисы:

### `finance-service`
Сервис хранения и обработки финансовых данных.

Отвечает за:
- пользователей;
- операции;
- категории;
- лимиты;
- рабочие пространства;
- участников общих бюджетов;
- импорт выписок.

Использует базу данных PostgreSQL.

### `analytics-service`
Сервис аналитики и формирования отчётов.

Отвечает за:
- дневные сводки;
- месячные отчёты;
- анализ расходов;
- экспорт данных в CSV и XLSX;
- данные для Mini App.

### `api-gateway`
Единая точка входа для внутренних и пользовательских запросов.

Отвечает за маршрутизацию запросов между сервисами.

### `bot-service`
Telegram-бот, предоставляющий основной пользовательский интерфейс.

### `Mini App`
Веб-интерфейс для расширенного просмотра данных пользователя.

Взаимодействие между сервисами осуществляется по HTTP с использованием внутреннего API-ключа.

## 6. Настройка окружения

Перед запуском проекта необходимо перейти в каталог `finance_helper/source_files`, создать файл `.env` на основе шаблона `.env.example` и указать реальные значения переменных окружения.

Обязательные параметры:

- `BOT_TOKEN` — токен Telegram-бота;
- `INTERNAL_API_KEY` — внутренний ключ взаимодействия сервисов;
- `MINIAPP_SIGNING_SECRET` — секретный ключ для подписи токенов Mini App;
- `MINIAPP_PUBLIC_URL` — публичный HTTPS-адрес Mini App;
- параметры подключения к базе данных PostgreSQL.

Пример:

```bash
cd finance_helper/source_files
cp .env.example .env
```

На Windows можно просто создать файл `.env` рядом с `docker-compose.yml` и скопировать в него значения из `.env.example`.

## 7. Установка зависимостей

Для локальной работы с проектом рекомендуется использовать виртуальное окружение Python.

Создание виртуального окружения:

```bash
python -m venv venv
```

Активация виртуального окружения:

Linux/macOS:
```bash
source venv/bin/activate
```

Windows:
```bash
venv\Scripts\activate
```

Установка зависимостей для тестов:

```bash
pip install -r tests/requirements.txt
```

Установка зависимостей для сервисов:

```bash
pip install -r services/finance-service/requirements.txt
pip install -r services/analytics-service/requirements.txt
pip install -r services/api-gateway/requirements.txt
pip install -r services/bot-service/requirements.txt
```

## 8. Запуск проекта

### 8.1 Подготовка
Необходимо убедиться, что установлены:
- Docker;
- Docker Compose.

### 8.2 Запуск
Выполнить команды:

```bash
cd finance_helper/source_files
docker compose up --build
```

После запуска будут подняты:
- PostgreSQL;
- `finance-service`;
- `analytics-service`;
- `api-gateway`;
- `bot-service`.

### 8.3 Остановка
Для остановки проекта выполнить:

```bash
cd finance_helper/source_files
docker compose down
```

Для остановки с удалением томов базы данных:

```bash
cd finance_helper/source_files
docker compose down -v
```
